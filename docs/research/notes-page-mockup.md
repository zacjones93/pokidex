## Layout Breakdown

### Left Column (40%)
- **Header**: Pokédex title + description
- **Pokemon Grid**: 3 columns x 7 rows (21 Pokemon per page)
- **Pagination**: Previous/Next + page numbers with loading spinners
- **Page Info**: "Page X of Y (N total Pokémon)"

### Right Column (60% - Sticky)
Split into 3 sections, each in a card:

1. **Pokemon Card** (`<PokemonCard>` component)
   - Pokemon sprite/image
   - ID, Name, Type badges
   - Height, Weight
   - Abilities list
   - Stats with progress bars

2. **Evolution Chain** (`<PokemonEvolutions>` component)
   - Visual evolution path
   - Evolution methods (level up, stones, etc.)
   - Current stage indicator

3. **Notes Section** (`<Outlet>` → `home-outlet.$id.notes.tsx`)
   - **Add Note Form**:
     - Textarea (4-5 rows)
     - "Add Note" button
   - **Notes List** (scrollable if many):
     - Note content
     - Timestamp
     - Delete button (×)
     - Newest first

## Responsive Behavior

### Desktop (lg+)
- Two columns side-by-side
- Right column sticky (follows scroll)
- Notes section scrollable independently

### Mobile (<lg)
- Single column stack
- Pokemon list → Pokemon card → Evolutions → Notes
- No sticky behavior

## Visual States

### Loading States
1. **Pokemon Card**: Skeleton animation while Pokemon detail loads
2. **Evolutions**: Spinner while evolution chain fetches
3. **Notes**: Suspense fallback skeleton (3 note skeletons) while IndexedDB loads
4. **Pagination**: Spinner next to clicked page number

### Empty States
1. **No Pokemon Selected**: Right side shows placeholder "Select a Pokémon to view details"
2. **No Notes**: "No notes yet. Add your first note about [Pokemon Name]!"

### Interaction States
- **Selected Pokemon**: List item has blue ring + scale effect
- **Hover**: Cards lift slightly
- **Active Note Delete**: Confirmation or instant delete with fade-out animation

## Dark Mode
All cards have dark variants:
- `bg-white dark:bg-gray-800`
- `text-gray-900 dark:text-gray-100`
- `border-gray-200 dark:border-gray-700`
